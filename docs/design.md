# 設計

## 選択肢

| Option | 内容 | 評価 |
| --- | --- | --- |
| A | UI先行で作る | 早いが検証の再現性が弱い |
| B | core判定と証跡を先に固める | 自動テスト、docs、QCDSに強い |
| C | 外部サービス連携を先に作る | 見栄えは出るがリスクが高い |

## 採用案

Option B を採用する。タブ、用途メモ、プロジェクト名を1カードで保存する。

## UI/UX方針

- 1画面で入力、検証、警告、次アクションを確認できる。
- エラーと warning を色だけに依存せず、項目名と理由で示す。
- 手戻りを避けるため、実行前に必要項目とリスクを見せる。
